# ADR Template

## Title
- Short decision title

## Status
- Proposed | Accepted | Superseded

## Context
- What problem is being solved
- Which user-visible or operational pain triggered it

## Decision
- The chosen approach
- The domain contract or interface that changes, if any

## Read/Write Path
- Write path touched
- Read path touched

## Tests
- App-local checks and focused regression tests added or updated

## Rollback
- What can be reverted safely
- What data or compatibility risk remains after rollback
